# SF4 Host — Backend + LLM Tone Engine + Web UI

A single FastAPI app that owns the USB serial link to the Arduino DSP amp, runs a
conversational OpenAI-first tone engine, and serves a two-panel web dashboard.

## Layout

```
app/
  main.py        FastAPI app: owns SF4Link + amp state; /api/* + serves the UI
  models.py      AmpParams (firmware-range Pydantic model), Telemetry, schemas
  serial_link.py SF4Link (real board) + MockLink (no-hardware) + autodetect
  tone_engine.py conversational LLM engine with the set_amp_params tool
  static/        BUILT React UI (generated by `make web-build` — gitignored)
web/             React + Vite frontend (source) — components, hooks, styles
sf4_serial.py    CLI test tool (sends one P frame, prints telemetry)
Makefile         automation: env / install / run / web-build / web-dev / cli / flash
requirements.txt pip dependency list (mirror of pyproject.toml)
.env.example     config template — copy to .env
```

The frontend is a **React + Vite** app in `web/`. `npm run build` (via `make web-build`)
compiles it into `app/static/`, which FastAPI serves — so the build output is generated and
gitignored; `web/` is the source of truth.

## Quick start (Makefile)

```bash
cd src/host
make install            # build the React UI + install Python deps
make env                # create .env from the template
#   → edit .env and set OPENAI_API_KEY to enable chat + voice
make run                # start the web app  →  http://127.0.0.1:8000
```

| Target | Does |
|--------|------|
| `make env` | `cp .env.example .env` (no-op if it already exists) |
| `make install` | build the UI (`web-build`) **and** install Python deps (`uv sync`, else venv + pip) |
| `make web-build` | `npm --prefix web install && npm run build` → `app/static/` |
| `make web-dev` | Vite dev server (:5173, hot reload, proxies `/api` to :8000) |
| `make run` | start the app on `$(HOST):$(PORT)`; builds the UI first if missing |
| `make dev` | same, with `--reload` |
| `make cli ARGS="--demo overdrive"` | bench-test the serial link |
| `make flash` | `arduino-cli` compile + upload the firmware (uses `FQBN` + `SF4_PORT`) |
| `make clean` | remove the venv, caches, and built UI |

The Makefile auto-detects `uv` (falling back to a plain venv) and loads `.env`.

### Frontend dev loop

For live UI work, run the API and the Vite dev server in two terminals:

```bash
make run        # terminal A — FastAPI on :8000 (serves /api + the built UI)
make web-dev    # terminal B — Vite on :5173 with hot reload, /api proxied to :8000
```

Edit `web/src/**` and changes hot-reload at <http://127.0.0.1:5173>. When done, `make
web-build` bakes it into `app/static/` so plain `make run` serves the latest.

## Run manually (without make)

```bash
cd src/host
npm --prefix web install && npm --prefix web run build   # build the UI → app/static
uv sync                               # or: pip install -r requirements.txt
cp .env.example .env                  # then set API keys / proxy URL in it
uvicorn app.main:app --reload         # .env is auto-loaded at startup
```

Open <http://127.0.0.1:8000>.

- **Hardware**: plug in the Uno; the port auto-detects (override with `SF4_PORT` in `.env`).
- **No hardware**: the backend falls back to a mock board — the UI and (with a key) the chat
  still work end to end, with synthetic telemetry.
- **No API key**: the manual control panel works; chat/voice return clear 503s.

### Configuration (`.env`)

`.env` is loaded automatically on startup (via `python-dotenv`). Copy `.env.example`
to `.env` (or run `make env`). `.env` is gitignored — never commit your key.

| Var | Default | Meaning |
|-----|---------|---------|
| `SF4_LLM_PROVIDER` | `openai` | tone engine provider (`openai` or `anthropic`) |
| `OPENAI_API_KEY` | — | required for OpenAI chat and voice input |
| `OPENAI_CHAT_MODEL` | `gpt-5.5` | strongest low-latency chat model for tone generation |
| `ANTHROPIC_API_KEY` | — | optional fallback when `SF4_LLM_PROVIDER=anthropic` |
| `SF4_MODEL` | `claude-haiku-4-5` | Anthropic model when using the Anthropic provider |
| `OPENAI_BASE_URL` | OpenAI default | set to your enterprise proxy URL if required |
| `OPENAI_TRANSCRIBE_MODEL` | `whisper-1` | speech-to-text model |
| `SF4_PORT` | auto-detect | force a serial port (e.g. `/dev/tty.usbmodem1101`) |
| `HOST` / `PORT` | `127.0.0.1` / `8000` | web server bind address (used by `make run`) |

## API

| Endpoint | Purpose |
|----------|---------|
| `POST /api/params` | manual control panel: apply a full `AmpParams` (no LLM) |
| `POST /api/chat` | one chat turn `{message, history}` → `{reply, params, changed}` |
| `POST /api/transcribe` | voice input audio blob → `{text}` |
| `GET /api/state` | current `AmpParams` (sync both panels) |
| `GET /api/telemetry` | SSE stream of live telemetry (~10 Hz) |

## CLI (bench test, no browser)

```bash
make cli ARGS="--demo overdrive"          # or, directly:
python sf4_serial.py --effect reverb --feedback 190 --mix 220
```

## Flashing the firmware

```bash
make flash          # needs arduino-cli; set SF4_PORT in .env first
                    # override the board with: make flash FQBN=arduino:avr:uno
```
